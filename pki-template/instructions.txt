🟩 INSTRUCTIONS FOR TESTERS — PK OFFICE SIMULATION

This folder contains a fully automated PKI environment for testing mTLS authentication with the military application.
The goal is to allow testers to act as a simplified Certificate Authority (CA) without manually configuring OpenSSL.

By following these steps, testers will be able to:

Create a Root CA

Create an Intermediate CA

Issue certificates for soldiers and commanders

Revoke certificates

Provide .crt files to the frontend/client for mTLS testing

1️⃣ Folder Structure Overview
pki-template/
│
├── setup-test-ca.sh       # Runs once — creates entire PKI
│
├── root-ca/               # Root Certificate Authority
│   ├── openssl.cnf
│   ├── serial             # contains: 1000
│   ├── index.txt          # empty
│   ├── certs/             # empty
│   ├── crl/               # empty
│   ├── newcerts/          # empty
│   └── private/           # empty
│
├── intermediate/          # Intermediate Certificate Authority
│   ├── openssl.cnf
│   ├── serial             # contains: 1000
│   ├── crlnumber          # contains: 1000
│   ├── index.txt          # empty
│   ├── certs/             # empty
│   ├── csr/               # empty — CSR files go here
│   ├── crl/               # empty
│   ├── newcerts/          # empty
│   └── private/           # empty
│
└── scripts/
    ├── issue-cert.sh      # Used to issue certificates
    └── revoke-cert.sh     # Used to revoke certificates


All directories are intentionally empty — the CA will populate them during operation.

2️⃣ Run the Setup Script (Only Once)

This script initializes the entire PKI:

Generates Root CA key + certificate

Generates Intermediate CA key + certificate

Signs the Intermediate CA

Creates the CA chain

Run:

cd pki-template
chmod +x setup-test-ca.sh
./setup-test-ca.sh


You should now see:

root-ca/private/root-ca.key

root-ca/certs/root-ca.crt

intermediate/private/intermediate-ca.key

intermediate/certs/intermediate-ca.crt

intermediate/certs/ca-chain.crt

The PK Office is now ready to issue user certificates.

3️⃣ Issuing a User Certificate

Users (soldiers/commanders) will provide a CSR file, generated by your desktop app or client tooling.

Place the CSR into:

pki-template/intermediate/csr/


Then run:

./scripts/issue-cert.sh intermediate/csr/USERNAME.csr


This produces:

intermediate/certs/USERNAME.crt


Return this .crt file to the user.
They will install it in their browser or OS keychain for mTLS authentication.

4️⃣ Revoking a Certificate (Optional)

If a certificate must be revoked (expired soldier role, key compromise, etc.):

./scripts/revoke-cert.sh intermediate/certs/USERNAME.crt


This updates:

intermediate/crl/intermediate-ca.crl


The backend can optionally use this CRL to reject revoked certificates.

5️⃣ What Testers Do NOT Need to Do

Testers do NOT need to:

Modify openssl.cnf files

Manually create keys

Manually sign certificates

Edit serial or index files

Understand OpenSSL internal details

Everything is automated.
